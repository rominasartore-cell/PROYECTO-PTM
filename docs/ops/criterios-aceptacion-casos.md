# PTM - Criterios de aceptacion de casos

## Casos aptos para flujo digital

- Certificado de multas de transito legible.
- Patente identificable.
- Fechas de ingreso al registro visibles.
- Multas antiguas con posible prescripcion.
- Cliente entiende que no hay resultado garantizado.
- Cliente completa datos postpago.

## Casos que requieren revision manual previa

- Certificado borroso, incompleto o cortado.
- Fechas no visibles.
- Patente inconsistente.
- Cliente no sabe si el certificado esta actualizado.
- Multas recientes.
- Muchas paginas o volumen alto.
- Caso con antecedentes judiciales adicionales.
- Cliente necesita representacion, no solo documentos.

## Casos que no conviene aceptar por flujo automatico

- Cliente exige garantia de resultado.
- Cliente quiere eliminar multas inmediatamente.
- Cliente no acepta que decide el tribunal u organismo competente.
- Cliente no entrega certificado.
- Cliente entrega imagen ilegible en vez de PDF claro.
- Caso requiere estrategia judicial personalizada.
- Cliente quiere patrocinio completo pero solo paga paquete digital.

## Regla operativa

Si el caso no se entiende con el certificado, no se fuerza el flujo automatico.

Mejor perder una venta que ganar un problema.
