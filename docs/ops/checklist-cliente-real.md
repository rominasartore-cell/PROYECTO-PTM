# PTM - Checklist cliente real

Usar este checklist por cada cliente pagado.

## 1. Pago

- [ ] Pago aparece aprobado.
- [ ] RequestId identificado.
- [ ] Correo cliente identificado.
- [ ] Patente identificada.

## 2. Datos postpago

- [ ] Cliente completo datos para documentos.
- [ ] RUT informado.
- [ ] Domicilio informado.
- [ ] Comuna informada.
- [ ] Profesion u oficio informado.
- [ ] Datos revisados antes de generar documentos.

## 3. Admin / JSON

- [ ] Solicitud visible en admin.
- [ ] Datos de analisis correctos.
- [ ] JSON admin disponible en carpeta tmp.
- [ ] RequestId coincide con cliente.

## 4. Generacion local

- [ ] Ejecutado generate-real-delivery.ps1.
- [ ] delivery-data.json generado.
- [ ] Informe generado.
- [ ] Instructivo generado.
- [ ] Escritos generados.
- [ ] HTML exportado.
- [ ] ZIP generado.

## 5. Control de calidad

- [ ] Validacion entrega APROBADO.
- [ ] Auditoria entrega APROBADO.
- [ ] Hallazgos 0.
- [ ] Revision visual realizada.
- [ ] No hay textos pendientes.
- [ ] No hay placeholders visibles.
- [ ] Rol de causa no se inventa si no viene informado.

## 6. Envio

- [ ] Correo de documentos listos enviado.
- [ ] ZIP o instrucciones entregadas.
- [ ] Registro interno actualizado.
